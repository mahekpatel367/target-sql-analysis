# Target SQL Analysis

## Project Overview
This project uses SQL to analyze retail data from Target to understand customer spending behavior, order patterns, and regional performance.

The analysis demonstrates the use of joins, subqueries, CTEs, and window functions to answer business-driven questions.

## Business Objective
- Analyze customer spending patterns
- Identify top customers and regions
- Evaluate order trends over time
- Support data-driven retail decisions

## Dataset
The dataset consists of multiple relational tables including:
- Customers
- Orders
- Order items
- Products
- Regions

## SQL Concepts Used
- Joins
- Subqueries
- Common Table Expressions (CTEs)
- Window Functions
- Aggregations

## Key Insights
- High-value customers contribute significantly to revenue
- Certain regions outperform others consistently
- Repeat customers drive long-term value
